ROTEIRO DE DEMO - SENTINELA SOC 4.0

1. Subir ambiente
Executar:
docker compose up -d --build

Verificar:
docker ps
docker compose logs --tail=120

2. Abrir dashboard
Acessar:
http://localhost:8080

Tambem validar API:
http://localhost:5000/alertas

3. Explicar arquitetura
Mostrar que o projeto usa:
simulator/log_collector -> Kafka raw_logs -> rule_engine -> Kafka security_alerts -> alert_sink -> PostgreSQL -> dashboard_api -> dashboard_web.

4. Mostrar eventos normais
No feed, apontar eventos NORMAL, HTTP_REQUEST ou DNS_QUERY com risco baixo. Explicar que trafego normal recorrente nao deve gerar resposta agressiva.

5. Mostrar ataque multiestagio
Apontar uma sequencia PORT_SCAN seguida de BRUTE_FORCE. Explicar que o rule_engine mantem estado em memoria por IP e classifica como ATAQUE MULTIETAPA com risco minimo 97.

6. Mostrar threat intel match
Usar um alerta de IP como 45.67.89.12, 185.220.101.44, 91.219.236.15, 103.27.202.66 ou 172.16.5.67. Mostrar threat_intel_match, categoria e descricao no feed. Explicar que a base local simula IOCs conhecidos.

7. Mostrar threat level subindo
Quando o dashboard detectar risco alto, mostrar o Threat Level mudando para HIGH ou CRITICAL e o banner INCIDENTE CRITICO EM ANDAMENTO.

8. Mostrar bloqueio simulado
Apontar a coluna Acao SOC com BLOQUEIO SIMULADO. Explicar que simulated_block=true e auto_response=simulated_block representam a decisao defensiva segura.

9. Explicar bloqueio real desativado
Deixar claro que ENABLE_BLOCK=false permanece como padrao, iptables real nao e usado e o projeto evita impacto real em rede por seguranca.

10. Encerrar com valor profissional
Concluir dizendo que o SENTINELA demonstra fundamentos de SOC, SIEM, Kafka, Python, PostgreSQL, Docker, threat intelligence, correlacao de eventos e visualizacao executiva em tempo real.
